### Как работает:

1. **Вместимость**: 20 записей (deque с автоудалением старых)
2. **Назначение**: Быстрый доступ к недавним данным
3. **Хранение**: `stm_{user_id}.json`

### Структура записи:

```python
@dataclass
class MemoryRecord:
    id: str                      # Уникальный ID
    timestamp: datetime          # Время создания
    content: str                 # Содержимое
    context: Dict               # Контекст
    importance: float           # Важность (0.0-1.0)
    access_count: int           # Количество обращений
    last_accessed: datetime     # Последнее обращение
    associations: List[str]     # Связи с другими записями
    memory_type: str            # "short_term" или "long_term"
    consolidation_score: float  # Оценка для консолидации
    emotional_valence: float    # Эмоциональная окраска
    sensory_data: Dict          # Дополнительные данные
```

## 🗄️ ДОЛГОВРЕМЕННАЯ ПАМЯТЬ

### Как работает:

1. **Назначение**: Постоянное хранение важной информации
2. **Формат**: JSONL (построчный JSON)
3. **Хранение**: `ltm_{user_id}.jsonl`

### Критерии консолидации:

Запись переносится из кратковременной в долговременную, если:

```python
consolidation_score >= 0.7
```

**Формула расчёта:**

```python
score = (
    importance * 0.3 +                    # Важность
    min(access_count/10, 1.0) * 0.3 +    # Частота обращений
    abs(emotional_valence) * 0.2 +       # Эмоции
    min(len(associations)/5, 1.0) * 0.2  # Связи
)
```

### Пример консолидации:

```
📦 Кратковременная: "Пользователь спросил о Python"
   ↓ (после 3 обращений, importance=0.8)
📚 Долговременная: Запись консолидирована!
```

### Алгоритм схожести:

```python
def _calculate_similarity(text1, text2) -> float:
    words1 = set(text1.lower().split())
    words2 = set(text2.lower().split())
    
    intersection = words1 & words2
    union = words1 | words2
    
    return len(intersection) / len(union)
```

## 💭 ОБРАТНАЯ ПРОГРЕССИЯ ДЛЯ ВООБРАЖЕНИЯ

### Концепция:

Вместо движения **вперёд** (от прошлого к будущему), система движется **назад** (от желаемого будущего к настоящему), чтобы найти путь достижения цели.

### Как работает:

```
ЦЕЛЬ: "Создать чат-бота"
         ↑
      [ШАГ 3] Протестировать бота
         ↑
      [ШАГ 2] Написать код обработчиков
         ↑
      [ШАГ 1] Изучить библиотеку python-telegram-bot
         ↑
   НАСТОЯЩЕЕ
```

### Пример использования:

```python
# Пользователь хочет достичь цели
goal = "создать веб-приложение на Flask"

# Система использует воображение
imagination_sequence = memory_system.reverse_progression_for_imagination(
    goal_state=goal,
    steps=5
)

# Результат: 5 шагов от настоящего к цели
# Каждый шаг основан на долговременной памяти
```

### Алгоритм:

1. Находим в долговременной памяти записи, релевантные цели
2. Выбираем наиболее подходящую
3. Идём назад по ассоциациям (к более ранним записям)
4. Строим обратную последовательность
5. Создаём "воображаемые" промежуточные состояния в кратковременной памяти

### Код:

```python
def reverse_progression_for_imagination(self, goal_state: str, steps: int = 5):
    # 1. Находим релевантные записи из ДВП
    relevant_ltm = self.recall_memory(goal_state, memory_type='long_term', limit=10)
    
    # 2. Строим обратную последовательность
    imagination_sequence = []
    current_memory = relevant_ltm[0]  # Начинаем с самой релевантной
    
    # 3. Идём назад по ассоциациям
    for _ in range(steps - 1):
        associated_memories = [
            self.memory_index[assoc_id]
            for assoc_id in current_memory.associations
            if assoc_id in self.memory_index
            and self.memory_index[assoc_id].timestamp < current_memory.timestamp
        ]
        
        if not associated_memories:
            break
        
        # Выбираем наиболее важную предшествующую запись
        current_memory = max(associated_memories, key=lambda x: x.importance)
        imagination_sequence.insert(0, current_memory)
    
    # 4. Создаём промежуточные "воображаемые" состояния
    for i, memory in enumerate(imagination_sequence):
        self.add_memory(
            content=f"[ВООБРАЖЕНИЕ ШАГА {i+1}] {memory.content[:100]}",
            context={'type': 'imagination', 'goal': goal_state},
            importance=0.6
        )
    
    return imagination_sequence
```

### 2. Создание ассоциаций

```python
# Автоматически связывает новую запись с последними 5
def _create_associations(self, record):
    recent_memories = list(self.short_term_memory)[-5:]
    
    for mem in recent_memories:
        similarity = self._calculate_similarity(record.content, mem.content)
        if similarity > 0.3:  # Порог
            record.associations.append(mem.id)
            mem.associations.append(record.id)
```

## 🐛 Решение проблем

### Бот не отвечает

1. Проверьте токен в `.env`
2. Проверьте подключение к интернету
3. Убедитесь, что бот запущен

### Ошибка "DDGS not found"

```bash
pip install duckduckgo-search
```

Или закомментируйте WebSearchCore, если поиск не нужен.

### Ошибка LM Studio

LM Studio опционален. Бот работает без него, используя только ядра.

Если хотите использовать LLM:
1. Установите LM Studio
2. Запустите локальный сервер
3. Проверьте URL в `.env`

## 🚀 Продвинутое использование

### Создание собственного ядра

```python
class CustomCore(KnowledgeCore):
    name = "custom_core"
    description = "Моё кастомное ядро"
    capabilities = ["что-то полезное"]
    priority = 5
    
    def can_handle(self, query: str) -> bool:
        return "моё слово" in query.lower()
    
    def execute(self, query: str, context=None) -> CoreResponse:
        # Ваша логика
        return CoreResponse(
            success=True,
            raw_result="Мой ответ",
            source=self.name,
            direct_answer=True
        )
```

Добавьте в `_load_builtin_cores()`:
```python
def _load_builtin_cores(self):
    builtin_cores = [
        DateTimeCore(),
        CalculatorCore(),
        CustomCore(),  # Ваше ядро
    ]
```
